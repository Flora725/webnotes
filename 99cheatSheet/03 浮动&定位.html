浮动规则
  • 1. 脱离标准文档流。在标准流中不占位置，覆盖在标准流之上。
  • 2. 浮动找浮动，不浮动找不浮动。
  • 3. 浮动的元素只会影响下面的元素，不会影响上面的元素，就好像是上面的非浮动的元素形成了一个挡板。
  • 4. 浮动的时候，内容（文字|图片）不会被覆盖，它会围绕浮动的元素呈现出来。
浮动元素的特点
  •     浮动的元素变成了inline-block元素：可以设置宽高(block)，如果不设置的话就由内容撑开(inline)；也可以排在一行(inline)，如果排不下的话就只好往下排。
清浮动
  • 方法一
    在父盒子底部添加一个空标签;
    缺点：会大量引入空标签
  • 方法二
    给父元素设置 overflow: hidden;
    触发BFC
  • 方法三
    给浮动元素下面的元素加上clear: both;
  • 方法四
    给父元素添加伪元素
    .father : after {
      content: “ ”;
      height: 0;
      display: block;
      clear: fix;
      visibility: hidden;
    }
相对定位relative
  • 参照物
    以原来的位置为参照物发生偏移
  • 滚动条
    随之滚动
  • 是否脱标
    不脱标。
    其他元素仍以原来的位置为基准正常注入。
  • 自身特点
    想当年
    没有脱离标准流。只是相对原来的位置进行了偏移。
绝对定位absolute
  • 参照物
    若无父元素，相对于body定位；有父元素，但父元素无定位，仍相对于body定位；有父元素且父元素有定位，则相对于父元素定位
  • 滚动条
    随之滚动
  • 是否脱标
    脱标。
    一个元素绝对定位时，会把它从页面的正常流中删除，在页面中不占任何位置
  • 自身特点
    找干爹
    会脱离标准流，在页面不占位置。对子盒子进行绝对定位的时候，一定要将父盒子进行相对定位，子绝父相
固定定位fixed
  • 参照物
    浏览器窗口，window
  • 滚动条
    不动
  • 是否脱标
    脱标。
  • 自身特点
    死心眼
    固定定位的元素会脱离标准流。
